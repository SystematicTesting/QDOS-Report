Welcome to QDOS (Quality Definition of System)

QDOS is an Open Source Test Automation Framework, which is designed on the principle of "Automate Testing without any Technical Knowledge".

Introduction

QDOS is developed on top of Selenium API and is used to automate testing for web sites on browsers like Firefox, Google Chrome, PhantomJS (Headless) etc. To use and integrate this framework, NO major technical skills required which means no programming knowledge required in any any of the language like Java, C#, .Net, Ruby etc.

QDOS Framework helps automate testing for web sites on market standard internet browsers. Main idea of this framework is to eliminate the technical skills required to automate any web site, which also means that any Manual Tester can use this framework and automate the testing of any web site.

"Manual Tester can be converted to Automation Tester via QDOS - framework."

QDOS Console Modules Features in QDOS Console

Reports

Reports module is responsible to display test reports of the automation test suite executed by the tester either on QDOS Grid (Enterprise License Product) or on Local Machine. This module not only just displays the reports, it also maintains them in form of version history. So, this module is very helpful to analyse the progress in test automation and also highlights the amount of test cases failed on every run.


Sites

Sites module is responsible to display test suites, test cases and their test steps in nice graphical format. This module not only just display the test data created by a tester, it also maintains the version history of all test data, test cases and test suites. This module facilitate tester to write test cases in Microsoft Excel Sheet and then upload them in QDOS Console either one by one or as bulk upload. Test Suites Uploaded here is always executed by QDOS Framework.


Site Manager

Site Manager module is responsible to manage access permissions on sites created inside Sites Module. It supports 3 ways of access permissions.

  • Public : Sites will be shared in read only mode to all the users of QDOS Console regardless of their Company.
  • Company : Sites will be shared in read only mode to the users which are part of SAME COMAPNY in QDOS Console Application.
  • Private : Sites will be completely private to user.
